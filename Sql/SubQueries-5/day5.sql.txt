# Why Window Functions?

Until now we solved problems using:

* Subqueries
* Correlated Subqueries
* GROUP BY

But interviewers often ask:

* Top 3 Salaries
* Top N Employees
* Second Highest Salary in Each Department
* Employee Ranking

Window Functions make these problems much easier.

---

# What is a Window Function?

A Window Function performs calculations across a group of rows while still returning every individual row.

Unlike GROUP BY:

* GROUP BY returns one row per group.
* Window Functions return every row with additional calculated information.

---

# ROW_NUMBER()

## Definition

Assigns a unique number to every row.

Even if two employees have the same salary,
their row numbers are different.

---

## Syntax

```sql
ROW_NUMBER() OVER
(
    ORDER BY salary DESC
)
```

---

## Example

| Employee | Salary | ROW_NUMBER |
| -------- | ------ | ---------: |
| Kiran    | 95000  |          1 |
| Deepa    | 90000  |          2 |
| Arjun    | 90000  |          3 |
| Rahul    | 70000  |          4 |

Notice

Two employees have salary 90000

Still

```text
2

3
```

Different numbers.

---

# RANK()

## Definition

Employees with the same salary receive the same rank.

The next rank is skipped.

---

## Syntax

```sql
RANK() OVER
(
    ORDER BY salary DESC
)
```

---

## Example

| Salary | Rank |
| ------ | ---: |
| 95000  |    1 |
| 90000  |    2 |
| 90000  |    2 |
| 85000  |    4 |

Notice

Rank 3 is skipped.

---

# DENSE_RANK()

## Definition

Employees with the same salary receive the same rank.

No ranks are skipped.

---

## Syntax

```sql
DENSE_RANK() OVER
(
    ORDER BY salary DESC
)
```

---

## Example

| Salary | Dense Rank |
| ------ | ---------: |
| 95000  |          1 |
| 90000  |          2 |
| 90000  |          2 |
| 85000  |          3 |

Perfect for interview questions involving
Top N Distinct Salaries.

---

# Difference Between the Three

| Salary | ROW_NUMBER | RANK | DENSE_RANK |
| ------ | ---------: | ---: | ---------: |
| 95000  |          1 |    1 |          1 |
| 90000  |          2 |    2 |          2 |
| 90000  |          3 |    2 |          2 |
| 85000  |          4 |    4 |          3 |

---

# PARTITION BY

## Definition

PARTITION BY divides the table into groups.

Ranking starts from 1 inside every group.

---

## Syntax

```sql
DENSE_RANK() OVER
(
    PARTITION BY departmentId
    ORDER BY salary DESC
)
```

---

## Example

Department 101

| Employee | Salary | Rank |
| -------- | ------ | ---: |
| Rahul    | 70000  |    1 |
| Priya    | 65000  |    2 |
| Sai      | 50000  |    3 |

Department 102

| Employee | Salary | Rank |
| -------- | ------ | ---: |
| Ravi     | 75000  |    1 |
| John     | 60000  |    2 |
| Anu      | 55000  |    3 |

Notice

Ranking restarts.

# Today's Interview Question

## LeetCode #185

Department Top Three Salaries

---

# SQL Solution

```sql
SELECT
    d.name AS Department,
    t.Employee,
    t.Salary
FROM
(
    SELECT
        name AS Employee,
        salary AS Salary,
        departmentId,
        DENSE_RANK() OVER
        (
            PARTITION BY departmentId
            ORDER BY salary DESC
        ) AS rnk
    FROM Employee
) t
JOIN Department d
ON t.departmentId = d.id
WHERE t.rnk <= 3;
```

## ROW_NUMBER()

Use when

Need exactly N rows.

---

## RANK()

Use when

Competition Ranking.

Use when

# Window Function Pattern

```sql
SELECT *
FROM
(
    SELECT *,
           DENSE_RANK() OVER
           (
               PARTITION BY group_column
               ORDER BY value DESC
           ) AS rnk
    FROM table_name
) t
WHERE rnk <= N;
```

Memorize this pattern.

# 🎯 Final Takeaway

Before choosing a Window Function, ask yourself:

**Do I need unique row numbers, competition ranking, or distinct salary ranking?**

* **ROW_NUMBER()** → Unique row numbers.
* **RANK()** → Competition ranking (skips ranks after ties).
* **DENSE_RANK()** → Top N distinct values (no skipped ranks).

Understanding this decision will help you solve many SQL interview questions quickly and correctly.
