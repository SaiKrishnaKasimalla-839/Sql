# Question 1

## Find employees who earn the highest salary in the company.

### Solution

```sql
SELECT emp_id,
       emp_name,
       salary
FROM Employee
WHERE salary =
(
    SELECT MAX(salary)
    FROM Employee
);
```

### Concept

* Scalar Subquery
* MAX()
* Entire Company

---

# Question 2

## Find employees whose salary is greater than the average salary.

### Solution

```sql
SELECT emp_id,
       emp_name,
       salary
FROM Employee
WHERE salary >
(
    SELECT AVG(salary)
    FROM Employee
);
```

### Concept

* Scalar Subquery
* AVG()

---

# Question 3

## Find employees whose salary is greater than the average salary of their own department.

### Solution

```sql
SELECT emp_name,
       salary,
       department_id
FROM Employee e
WHERE salary >
(
    SELECT AVG(salary)
    FROM Employee
    WHERE department_id = e.department_id
);
```

### Concept

* Correlated Subquery
* AVG()
* Department-wise Comparison

---

# Question 4

## Find the department having the highest average salary.

### Solution

```sql
SELECT department_id,
       AVG(salary) AS avg_salary
FROM Employee
GROUP BY department_id
ORDER BY avg_salary DESC
LIMIT 1;
```

### Concept

* GROUP BY
* AVG()
* ORDER BY
* LIMIT

---

# Interview Solution (Subquery)

```sql
SELECT department_id,
       AVG(salary) AS avg_salary
FROM Employee
GROUP BY department_id
HAVING AVG(salary)=
(
    SELECT MAX(avg_salary)
    FROM
    (
        SELECT AVG(salary) AS avg_salary
        FROM Employee
        GROUP BY department_id
    ) dept_avg
);
```

# Today's Doubt 2

## Why Correlated Subquery?

Question

Find employees earning maximum salary in their department.

For Rahul

```text
Department = 101

↓

Maximum Salary of Department 101

↓

70000

↓

Rahul Salary = 70000

↓

TRUE
```

For Ravi

```text
Department = 102

↓

Maximum Salary = 75000

↓

TRUE
```

Every employee compares against a different department average/maximum.

Therefore,

Use Correlated Subquery.

# Today's Doubt 3

## Company-Level vs Department-Level

### Company-Level

One comparison value for everyone.

Example

```sql
SELECT *
FROM Employee
WHERE salary=
(
SELECT MAX(salary)
FROM Employee
);
```

Everyone compares against

```text
95000
```

Only one comparison.

---

### Department-Level

Every employee compares with a different value.

Example

```sql
WHERE department_id=e.department_id
```

Department 101

↓

70000

Department 102

↓

75000

Department 103

↓

95000

Different comparison values.

Therefore,

Correlated Subquery.

# Common Mistakes

### Wrong

```sql
WHERE department_id IN
(
SELECT MAX(salary)
FROM Employee
GROUP BY department_id
);
```

Reason

Comparing

```text
department_id

with

salary
```

Different columns.

---

### Wrong

```sql
SELECT AVG(salary)
FROM Employee
GROUP BY department_id
```

Used with

```sql
salary >
(
subquery
)
```

Reason

Subquery returns multiple rows.

---

### Correct

Use Correlated Subquery

```sql
WHERE salary >
(
SELECT AVG(salary)
FROM Employee
WHERE department_id=e.department_id
);
```

---

# Interview Tips

✅ Entire Company → Simple Subquery

✅ Each Department → Correlated Subquery

✅ Department Result → GROUP BY

✅ Employee Result → Correlated Subquery

Always ask yourself:

> "Am I comparing everyone with one value, or is each row compared with a different value?"

If one value → Scalar Subquery

If different values → Correlated Subquery
